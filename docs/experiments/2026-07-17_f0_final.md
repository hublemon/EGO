# F0 final — WM-only(GT-free) GRPO 결과 종합

2026-07-17 · Step 2 (VLM Alignment) · F0 forward 트랙
코드: `src/ego/step2_vlm_alignment/` · 설정: `configs/step2/f0_final_wm_only.yaml`

## 0. 한 줄 결론

**입력 포맷 하나가 학습보다 7배 큰 효과를 냈다.** GRPO는 정확도를 +0.028 올렸지만, 그 대가로
**belief·history 추론이 오히려 퇴화**했다.

---

## 1. F0가 푸는 문제

Qwen3-VL-8B를, **정답(GT) 없이** V-JEPA2 world-model의 likelihood만으로 GRPO 정렬한다.

```
입력  : 앵커 프레임 + action history + WM의 joint action 후보 5개 (점수 숨김·순서 셔플)
출력  : <reasoning>…</reasoning> <task_belief>…</task_belief> <action>{"verb","noun"}</action>
리워드: WM likelihood 단독 (+ 구조 게이트).  GT 신호 0.
```

점수를 숨기고 순서를 셔플하므로 **어느 것이 WM top-1인지 식별할 수 없다** → 복사가 불가능하고,
프레임·history로부터 **WM이 무엇을 고를지 추론**해야 한다. 이것이 F0의 과제 전부다.

---

## 2. 상한선 — 모든 목표의 출발점

held-out n=500 실측 (V-JEPA2 ViT-G/384 + EK100 probe):

| 항목 | 값 |
|---|---|
| **GT action ∈ joint top-5 = 논리적 상한** | **0.620** |
| **joint top-1 = GT = 실질적 상한 (WM 완전 추종)** | **0.374** |
| **G2 구간 = 0.620 − 0.374** | **0.246** (123/500) |
| GT가 top-5에도 없음 — 원리적으로 불가 | 0.380 |
| joint top-5 균등선택 | 0.124 |

- **0.374** = "WM top-1을 그냥 베끼기"의 성적. 학습 없이 얻는 값.
- **0.620** = 재랭킹으로 도달 가능한 최대치.
- **0.246** = F0+B0가 벌 수 있는 전부.

## 3. 결과

### 3.1 요인 분해 — 이것이 이 실험의 핵심 산출물

F0 final은 이전 baseline(Run 1) 대비 **포맷·모델·태그·task_goal이 한꺼번에** 바뀌었다.
학습 없이 eval만으로 두 요인을 분리했다:

| | acc | 차이 | 뜻 |
|---|---|---|---|
| Qwen2.5 base @ 25조합 (verb-5 × noun-5) | 0.044 | — | 이전 baseline의 출발점 |
| **Qwen2.5 base @ joint top-5** | **0.248** | **+0.204** | **순수 포맷 효과** |
| Qwen3 base @ joint top-5 | 0.230 | **−0.018** | **순수 모델 효과 (음수)** |
| **Qwen3 step500 @ joint top-5** | **0.258** | **+0.028** | **학습 효과 (G1)** |

**읽는 법 세 가지:**

1. **포맷이 거의 전부다 (+0.204).** 학습을 하나도 안 한 Qwen2.5가 joint 포맷만으로 **0.248**을 낸다 —
   이전 baseline의 **완전 학습본(0.164)을 크게 앞선다.** 이전 트랙에서 GRPO가 벌던 +0.120은
   상당 부분 **나쁜 입력 포맷을 보상하고 있었을 뿐**이다.
2. **모델 업그레이드는 손해였다 (−0.018).** Qwen3-VL-8B < Qwen2.5-VL-7B.
3. **학습 기여는 작다 (+0.028).** 이전 포맷에서 +0.120이던 것이 포맷을 고치자 +0.028로 줄었다.

### 3.2 전체 지표

| run | acc | **G2** | in_joint5 | acc\|in5 | rank1\|in5 | wm_follow | think단어 |
|---|---|---|---|---|---|---|---|
| Qwen2.5 base @joint | 0.248 | **0.333** | 0.950 | 0.256 | 0.352 | 0.334 | 76.5 |
| Qwen3 base @joint | 0.230 | 0.301 | 0.982 | 0.231 | 0.327 | 0.321 | 86.8 |
| Qwen3 step125 | 0.228 | 0.301 | 0.990 | 0.232 | 0.332 | 0.329 | 90.6 |
| Qwen3 step250 | 0.252 | 0.333 | 0.990 | 0.256 | 0.345 | 0.342 | 90.7 |
| **Qwen3 step500** | **0.258** | **0.309** | 0.988 | 0.262 | 0.356 | 0.351 | 94.0 |

### 3.3 ★ G2가 처음으로 우연을 넘었다

**G2 = 0.309 (step500), 0.333 (Qwen2.5 base) vs chance 0.20.**

G2는 "WM top-1이 틀렸고 GT는 top-5에 있는" 구간(123/500)의 정확도다. 5지선다이므로 우연은 0.20,
"틀린 top-1을 안다고 가정"해도 0.25다. **0.31~0.33은 둘 다 넘는다** — WM이 틀린 곳에서
우연 이상으로 맞히고 있다. 이전 포맷에서는 G2가 0.155~0.171(25조합 우연 0.04의 4배였지만
5지선다 기준으로 환산 불가)이었다.

**⚠️ 단, 학습이 G2를 올린 것이 아니다.** Qwen3 base 0.301 → step500 0.309 (+0.008).
같은 구간에서 `wm_follow`는 0.321 → 0.351로 올랐다. **WM 모방이 늘어난 만큼 G2는 제자리**다.
이건 예측된 트레이드오프다 — 리워드가 "WM을 맞혀라"이므로 최적 정책은 WM 추종이고,
WM 추종은 정의상 G2 구간에서 손해다.

### 3.4 in_joint5 — 포맷 전환의 직접 증거

이전 포맷에서는 모델 출력의 **52.6%가 WM이 제안한 적 없는 조합**이었고 그 구간 정확도는 **0.027**이었다
(출력 질량의 절반이 버려짐). joint 포맷에서 **in_joint5 = 0.95~0.99**로 이 누수가 구조적으로 사라졌다.

> 참고: 이전 트랙의 `candidate_escape_rate`(0.008)는 이 누수를 **전혀 못 잡았다** —
> verb∈verb-top5 AND noun∈noun-top5라는 25조합 공간만 검사했기 때문이다. `in_joint5`가 옳은 지표다.

## 4. ★ 리즈닝 품질 — 외부 judge 곡선

내부 지표(lift/mask/shuffle)는 **품질을 못 잰다**: lift는 전 run 24 근처 포화, mask는 lift와 사실상 동일,
shuffle은 "의미 있는 추론"과 "도구적 인코딩"을 구분하지 못한다(둘 다 하락). 그래서 **외부 모델**에게
루브릭 채점을 받아 **학습 step에 따른 곡선**을 만들었다.

- judge: `claude-sonnet-4-6` · 25 step마다 · 생성 3개 · 20지점(step 1~476) · **비용 $0.055**
- **judge에게 GT를 주지 않는다** — 주면 '품질'이 아니라 '정답률'을 재게 되어 acc와 중복된다.
- **학습에 일절 미사용.** 롤아웃을 재사용하므로 GPU 비용 0, 학습 루프 밖에서 실행.

| 항목 (0~2) | 전반부 | 후반부 | 변화 |
|---|---|---|---|
| `candidate_review` (후보를 명시적으로 견주는가) | 0.80 | 1.40 | **+0.60** |
| `visual_grounding` | 1.35 | 1.60 | +0.25 |
| `no_confabulation` | 1.75 | 2.00 | +0.25 |
| `conclusion_follows` | 1.95 | 2.00 | +0.05 |
| **`history_grounding`** (history를 실제로 쓰는가) | 2.00 | 1.83 | **−0.17** |
| **`belief_globality`** (목표가 전역적인가) | 2.00 | 1.55 | **−0.45** |
| **총점 (0~12)** | 9.85 | 10.38 | **+0.53** |

### 4.1 이 표가 말하는 것 — 선택적 위축

**총점은 올랐지만 오르는 항목과 내리는 항목이 갈렸다.**

- **오른 것**: `candidate_review` (+0.60). GRPO가 정확히 이걸 가르쳤다 — 후보를 잘 견주면 WM top-1을 잘 맞힌다.
- **내린 것**: `belief_globality` (−0.45), `history_grounding` (−0.17).

**메커니즘은 명확하다: WM은 action history를 못 본다** (비디오 클립만 입력).
따라서 **history 추론과 전역 목표 추론은 "WM top-1 맞히기"에 도움이 안 된다** →
WM likelihood 리워드는 이를 **정확하게 깎아낸다**. Goodhart가 "임의 문자열"로 나타난 게 아니라
**도움 안 되는 부분의 선택적 위축**으로 나타났다.

**함의**: WM-only 리워드는 **B0가 필요로 하는 바로 그 추론(belief·history)을 침식한다.**
F0를 B0의 trace proposer로 쓰려면 이 상충을 설계에 반영해야 한다.

**한계**: 20지점 × 3생성, judge 1종. 시사적이지 결정적이지 않다. 하강은 후반 5지점(376~476)이 주도한다.
다른 judge 모델(gemini-2.5-pro 등)로 교차검증이 필요하다.

## 5. 확정된 설계와 그 근거

| 항목 | 설정 | 근거 (전부 실측) |
|---|---|---|
| 입력 포맷 | **joint action top-5** | 논리 상한 손실 −4.6pp(0.666→0.620)뿐인데 포맷 효과 **+0.204** |
| 후보 노출 | `--hide_scores` + 셔플 | 점수 노출 시 top-1 복사로 즉시 퇴화 (G2=0인 퇴화 정책) |
| 추론 태그 | **`<reasoning>`** | Qwen3-VL에서 `<think>`/`</think>`는 **예약 단일토큰**(151667/151668)이고 Instruct 변형은 절대 생성하지 않음 → `think=""` → **P4 영구 0**. `<reasoning>`으로 바꾸자 format 만점·P4 0.175로 정상화 |
| `num_generations` | **8** | 4는 `frac_reward_zero_std` **0.5~0.75**(그룹 분산 붕괴 → gradient 0). 8에서 0 |
| `temperature` | **1.0** | 1.2는 **쓰레기 토큰** 생성 (`LEGAL`, ` ilaç`, 프롬프트 placeholder 복사). entropy 상승 ≠ 유용한 다양성 |
| `max_completion_length` | **384** | Qwen3-VL은 verbose → 256에서 `clipped_ratio` 0.25 |
| 추론 길이 제한 | **없음** | clipping은 384가 해결. 길이를 손으로 묶으면 "리즈닝이 개선되는가"라는 질문 자체가 편향됨 |
| `max_steps` | **500** | `save_steps`(125)의 배수여야 최종 체크포인트가 생성됨 |
| dynamic sampling (런타임) | **off** | ablation에서 붕괴 없음 → 과잉 |
| P1 sharpening | **미사용** | 0.122 vs baseline 0.124 → 무효 |
| P3 (ref 모델 채점) | **미사용** | **WM이 아닌 신호**라 원칙상 배제 (성능 때문이 아님 — 가중 2.4%로 사실상 미검정이었다) |
| `task_goal` 필드 | **프롬프트에서 제거** | 실측: 비디오당 1종(=video ID) · GT 예측력 **1.1%** · 샘플은 중앙값 9.1분 뒤 시점 |

### 5.1 절대 금지 — 리워드에 WM 이외 신호

```
reward = WM likelihood × history-consistency   ← 금지
reward = WM likelihood + belief-quality        ← 금지
reward = WM likelihood + reasoning-score(P3)   ← 금지
```
셋 다 **사람이 손으로 정의한 휴리스틱**이지 WM에서 나온 신호가 아니다.
넣는 순간 G1("GT-free WM 신호만으로 학습이 성립한다") 주장 자체가 오염된다.
**Action History의 올바른 위치는 리워드가 아니라 입력이다.**

## 7. 알려진 한계

1. **memory가 GT 내레이션 유래.** "정답"이 아니라 "이미 일어난 일"이지만 엄격한 GT-free 주장엔 흠.
   self-memory(모델 자신의 과거 예측) 실험 미실시.
2. **held-out이 크로스도메인이 아님.** video-disjoint(겹침 0)이지만 참가자는 양쪽 동일(P01–P06).
   "도메인 범용성" 주장에는 미달.
3. **EK100만 검증.** Ego4D 미실시. 이식 시 **백본이 다르다** — EK100은 ViT-G/384, Ego4D LTA probe는
   ViT-L(`embed_dim=1024`)이라 probe가 구조적으로 ViT-L에 묶여 있다. 두 데이터셋 비교에 백본 교란이 낀다.
4. **judge 곡선의 표본이 작다** (20지점 × 3생성, judge 1종).
5. **`--reward_mode`를 학습과 평가에서 반드시 일치**시켜야 한다. 어긋나면 프롬프트 분포가 달라져 무의미한 숫자가 나온다.
