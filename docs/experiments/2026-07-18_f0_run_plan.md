## 1. 사전 검증 배터리 (handoff §10.1) — 실행 범위와 1차 결과

전부 학습 불필요·GT는 평가에만 사용. 러너: `scripts/step2/eval_battery.py`
(v2 프롬프트 빌더 `build_joint_conversation`를 직접 사용 — 기존 `evaluate.py`는 v1 플랫 스크립트를
import해 4f grid 안내를 모르므로 배터리 ⑤에 사용 불가). 조건: greedy·384tok·n=500 (v1 heldout 동일 표본).

| # | 실험 | 상태 | 1차 결과 (인코딩 크래시 전 로그 회수분) |
|---|---|---|---|
| — | 1f-base (strict cutoff) | 파이프라인이 재실행 중 | acc **0.242** · G2 0.309 · wm_follow 0.328 · in_joint5 0.986 |
| ① | no-memory | 〃 | acc **0.234** (히스토리 기여 **+0.8%p**) · G2 0.333 |
| ② | history-only (프레임 마스킹) | 라운드 2에서 실행 | — |
| ⑤ | 4f-base (관측 비대칭 해소) | 라운드 2에서 실행 | — ← **채택 게이트: acc > 0.30** |
| ③ | belief-swap | **미구현** — 본 파이프라인 범위 밖 | B0 착수 전 필수 (handoff §10.1) |
| ④ | judge `belief_action_link` | **미구현** — API 키·루브릭 추가 필요 | 롤아웃 250개 parquet은 이 서버에 있음 |

1차 관찰 (확정은 재실행본 기준):
- **strict cutoff 재평가**: 1f-base 0.242 vs v1 legacy 0.230 (+1.2%p) — cutoff 수정(heldout 64/1417=4.5% 수정, handoff의 4.6% 실측과 일치)이 base를 소폭 올림. 참조선(wm_top1 0.374 · oracle 0.620 · G2 n=123)은 동일 → 동일 표본 확인.
- **① 히스토리 기여 +0.8%p로 미미**: base가 히스토리를 최종 선택에 거의 안 씀 — judge의 history_grounding 만점(2.00)이 "reasoning에 인용은 하지만 acc에는 기여 없음"이었다는 해석을 지지. no-memory에서 G2가 오히려 상승(0.333)한 것도 히스토리가 G2 구간에서 순기여가 아님을 시사.

크래시 사후 기록: 평가 저장부의 ascii 인코딩 버그(수정 완료, `eval_battery.py` utf-8 고정). 생성은 greedy 결정론이라 재실행 수치는 동일할 것으로 예상.

## 4. 게이트 시나리오 — **판정 결과 (09:39 갱신)**

**⑤ 미달 확정: 4f-base acc 0.226 ≤ 0.30** (1f-base 0.242 대비 −1.6%p — 2×2 grid 합성이 base에서
오히려 손해). 담당자 결정 = **(b) 1f 유지**. 1f-strict train 셋(backup 프레임 재사용, 4,998샘플)
leakage PASS 후 09:39 500-step run(1f) 시작. 4f는 학습-결합 ablation으로 이월.
배터리 최종 수치: 1f-base 0.242 / ①no-mem 0.234(히스토리 기여 +0.8%p) / ②hist-only 0.210(우연+1%p) /
⑤4f 0.226. 부수 수정: accelerate 콘솔 스크립트 shebang이 구 서버 경로라 `python -m
accelerate.commands.launch` 호출로 교체(`train_pro_final_v2.sh`), `NUM_FRAMES` env 지원 추가.

### (원래 시나리오 기록)

- **⑤ 통과 (4f-base > 0.30)**: 그대로 500-step 진행 (지금 설정). 4f 입력 계약 확정 → full run도 4f.
- **⑤ 미달**: `NEEDS_DECISION` 파일에 옵션 기록 후 학습 전 정지. 선택지 — (a) `FORCE_4F=1` 강행,
  (b) 1f 유지로 500-step (입력 계약 변경), (c) 프레임 수/해상도 재설계. plan이 "채택 게이트"로 못 박은
  연구 결정이라 자동 진행하지 않는다.
- **leakage FAIL**: 즉시 정지. strict cutoff 구현 오류이므로 수정 전 학습 금지 (freeze 게이트, plan §0-4).

3. **r64 ablation** — 4f 효과 확정 후 별도 실행 (plan §0-3, 동시 변경 금지).
