## 1. F0가 검증하려던 것

Qwen VLM을 **정답(GT) 라벨 없이** V-JEPA2 world-model(WM)의 likelihood만으로 GRPO 정렬한다.

```
입력  : 앵커 프레임 1장 + action history + WM joint action 후보 5개 (점수 숨김·순서 셔플)
출력  : <reasoning>…</reasoning> <task_belief>…</task_belief> <action>{"verb","noun"}</action>
리워드: WM likelihood 단독 (+ 구조 게이트). GT 신호 0.
```

점수를 숨기고 순서를 셔플하므로 top-1 베끼기가 불가능하다 — 프레임·history로부터
**WM이 무엇을 고를지 추론**해야 한다. 검증 질문 세 가지:

- **Q1** GT 없이 WM 신호만으로 학습이 성립하는가
- **Q2** WM이 틀린 곳에서도 우연보다 잘 맞히게 되는가
- **Q3** 그 과정에서 리즈닝 품질이 좋아지는가

## 2. 용어

| 용어 | 뜻 |
|---|---|
| **acc (정확도)** | held-out 500샘플에서 최종 `<action>`의 (verb,noun)이 GT와 정확 일치한 비율. GT는 평가에만 사용 |
| **기여** | 요인 분해 표에서 바로 윗행과의 acc 차이. 한 번에 하나만 바꾸는 통제 비교라 그 요인의 순수 효과 |
| **G1** | 학습 효과 — 같은 모델·포맷에서 GRPO 전후 acc 차이 |
| **G2** | WM top-1이 틀렸지만 GT가 top-5 안에 있는 구간(123/500)의 acc. WM보다 나은 판단의 지표 |
| **wm_follow** | 모델 선택이 WM top-1과 일치한 비율 |
| **in_joint5** | 출력이 제시된 후보 5개 안에 든 비율 |

## 3. 요인 분해 — 핵심 결과 ①

| 무엇을 했나 | acc | 기여 |
|---|---|---|
| Qwen2.5 base @ 25조합(verb5×noun5), 학습 전 | 0.044 | 출발점 |
| **포맷만 joint top-5로 교체 (학습 0)** | **0.248** | **+0.204 ← 포맷 효과** |
| 모델을 Qwen3-VL-8B로 교체 | 0.230 | −0.018 (손해) |
| **GRPO 500 step 학습** | **0.258** | **+0.028 ← 학습 효과(G1)** |

- 포맷이 거의 전부다. 이전 트랙에서 GRPO가 벌던 +0.120은 나쁜 포맷을 보상하던 것.
- 모델 업그레이드는 손해 (Qwen3-VL-8B < Qwen2.5-VL-7B).
- G1 = +0.028로 작지만 0이 아님 → Q1 "부분 성립". 왜 작은지는 §9 진단 참조.

## 4. 상한선 구조 (held-out n=500, ViT-G/384 + EK100 probe)

| 구간 | 비율 | 뜻 |
|---|---|---|
| WM top-1 = GT | 37.4% (187) | 추종하면 득점. **실질적 상한 0.374** = 이 구간 전부 |
| G2 구간 | 24.6% (123) | 추종하면 실점. WM을 이겨야 버는 구간 |
| GT가 top-5 밖 | 38.0% (190) | 원리적으로 불가. **논리적 상한 0.620** = 위 둘의 합 |

현재 0.258은 실질 상한의 69% 지점. G2는 0.31~0.33으로 5지선다 우연(0.20)을 처음 돌파했으나
(Q2 부분 달성) **학습이 아니라 포맷의 공**이다: 학습 전 0.301 → 후 0.309, 같은 기간 wm_follow가
0.321 → 0.351로 오른 만큼 상쇄. 리워드가 "WM을 맞혀라"인 이상 예측된 트레이드오프.

### 6.1 judge 모델 정책 — 이후 Gemini 강제

**이후의 모든 judge 채점(Ego4D 곡선·belief_action_link 신규 항목·교차검증 재채점)은
gemini-2.5-pro로 통일하고 Claude 계열은 쓰지 않는다.**

- 이유 ①: 개발·분석 파이프라인이 Claude 계열 → 같은 계열 채점은 자기선호 편향 의혹에서 자유롭지 못함.
  판정 모델은 저자 측 도구와 계열을 분리한다.
- 이유 ②: 회의에서 25 step 모니터링 기준으로 삼은 모델도 Gemini.
- 비용: 호출당 $0.0024 (claude-sonnet-4-6의 25배, reasoning 토큰 소모)이지만 곡선 전체 ~$1.4로 감내 가능.
- 실행: `judge_reasoning.py --judge_model gemini-2.5-pro` (스크립트 기본값이 이미 gemini-2.5-pro).
  `configs/step2/f0_final_wm_only.yaml`의 `judge.model`을 gemini-2.5-pro로 변경할 것.
- 기존 claude-sonnet-4-6 곡선은 측정 기록으로 유지하고, 첫 작업으로 belief 퇴화를 Gemini로
  재채점해 일치 여부 확인 (현재 곡선은 20지점×3생성×judge 1종 — 시사적, 비결정적).

## 7. 히스토리 미사용의 원인 — EK100 탓인가?

- **주원인은 최적화 압력 (직접 증거 있음)**: 학습 전 base가 history_grounding 만점(2.00).
  EK100에서 히스토리가 본질적으로 무용했다면 base부터 낮았어야 한다. 원래 쓰던 능력을 학습이 깎았다 —
  목표가 GT가 아니라 **히스토리를 못 보는 WM의 선택**이기 때문.
- **EK100 특성은 부분 기여 가설**: *전역* 태스크 일관성은 실제로 약함 (task_goal GT 예측력 1.1%,
  샘플은 첫 내레이션에서 중앙값 9.1분 뒤). 반면 *국소*(직전 행동) 예측력은 높음 — cutoff 버그로 새던
  직전·중첩 행동의 예측력이 가장 높다는 실측이 방증.
- 분리 검증은 §10 사전 검증 배터리 ①②로. "히스토리를 쓰게 만들기"는 F0 리워드로는 불가능(금지 목록) —
  **B0의 hindsight projection DPO가 담당** (GT 기준에서는 히스토리 쓴 추론이 실제로 이김).

## 8. belief의 지위 — 생성되지만, 채점도 인과 측정도 없다

- belief는 매 샘플 생성되며 출력 순서상 action보다 **앞** → 인과 경로는 열려 있음.
- 그러나 ① belief를 채점하는 리워드가 없고 (의도된 설계 — WM-외 신호 금지),
  ② **belief→action 인과성을 재는 지표도 없다** (judge conclusion_follows는 reasoning→action만,
  내부 지표 lift/mask/shuffle도 reasoning 블록 대상).
- 실측 시사: belief_globality 퇴화의 패턴("belief = 선택 action의 재진술")은 **인과 역전** —
  action이 reasoning에서 결정되고 belief는 사후 요약 라벨. Embodied Reasoning이 요구하는 방향의 정반대.
- 확인 방법은 §10 배터리 ③④. **B0의 성공 판정 기준이 "belief가 action을 실제로 조향하게 됐는가"이므로
  F0 시점 baseline이 지금 필요하다.**

## 9. 진단 — 학습 기여가 +0.028에 머문 이유

1. **학습량이 1 epoch 미만 (가장 확실)**: TRL GRPO에서 `per_device_train_batch_size=8`은 생성 단위 →
   `num_generations=8`이면 GPU당 프롬프트 1개/step. 2 GPU × 500 step = **프롬프트 방문 1,000회 =
   데이터 5,000개의 20%를 한 번 본 것**. `gradient_accumulation_steps=1`이라 스텝당 그래디언트가
   프롬프트 2개분 — 극도로 노이지. **acc(0.228→0.252→0.258)·wm_follow(0.329→0.342→0.351) 모두
   단조 상승 중, 플래토 없음** — 수렴이 아니라 예산 때문에 멈춘 학습.
2. **관측 비대칭 (구조적)**: 리워드 심판 V-JEPA2는 4초×32프레임 클립을 보는데
   (`vjepa_infer_train.py`), 정책 VLM은 trigger_frame JPEG **1장**만 본다 (`extract_frame_train.py`).
   WM이 본 것의 1/32만 보여주며 WM의 선택을 맞히라는 구조 — rank1|in5가 0.36에 묶인 유력한 이유.
   §10 배터리 ⑤로 학습 없이 검증 가능.
3. **갱신 용량 (보조)**: LoRA r16, attention 4종만 — MLP·비전 계층 동결. `min_wm_spread=0.05`는
   약한 필터라 advantage≈0 샘플 다수. 원인 1 해소 후에야 병목으로 드러날 항목.

### 9.1 0.374까지의 산수

```
0.258 = 0.374 × 0.49 + 0.246 × 0.309   →   목표: 0.374 ≈ 0.374 × 0.85 + 0.246 × 0.25
```

현재 모델은 WM이 맞는 구간에서 **49%만 일치** (학습 전 42% → 49%로 상승 중).
G2를 우연 수준(0.20~0.25)으로 유지한 채 이 일치율을 0.85 안팎으로 올리면 0.374에 닿는다.
**도달을 막는 벽은 현재 관측되지 않았다** — 부족한 것은 학습량과 입력 정보.

## 10. 최종 개선 방향 (확정)

### 10.1 ★ 재학습 전 사전 검증 배터리 — 전부 학습 불필요, 합쳐서 GPU 반나절

| # | 실험 | 확정되는 것 | 비용 |
|---|---|---|---|
| ① | `--no_memory` 평가 (플래그 구현됨) | 히스토리 제거 시 acc·G2 하락폭 = 히스토리의 실제 기여량 | 평가 1회 |
| ② | history-only 베이스라인 (프레임 없이 5지선다) | 히스토리 단독 예측력. Ego4D Phase 0에서 재측정 시 데이터셋 효과 분리 | 평가 1회 |
| ③ | **belief-swap 개입 테스트** — 생성된 belief를 다른 전역 목표로 교체 후 그 지점부터 action만 재생성 | action 분포 변화 여부 = **belief→action 인과성 baseline**. 불변이면 belief는 인과적으로 무력 | 추론 1회분 |
| ④ | **judge 루브릭에 `belief_action_link` 추가** — "선택 action이 stated belief의 진행 단계로 정당화되는가" | belief→action 링크의 step 곡선. 롤아웃 재사용, GPU 0. judge = gemini-2.5-pro | API ~$1.4 |
| ⑤ | 클립(멀티프레임) 입력 base 평가 — 앵커 1장 → WM과 동일 4초 구간 8프레임 | 관측 비대칭 해소 시 rank1\|in5·wm_follow 상한 상승분 → 재학습에 클립 입력 채택 여부 결정 | 평가 1회 |

**③④는 B0 성공 판정의 사전값 — B0 착수 전에 반드시 확보하고 결과를 B0 담당자에게 전달한다.**

### 10.3 Ego4D × V-JEPA2 이식 (우리 담당)

- **Phase 0 — 상한선 게이트**: 학습 전 probe 추론만으로 Ego4D 상한선 테이블(논리·실질·G2 폭) 실측.
  배터리 ①②도 같은 held-out에서 재측정 → 데이터셋 효과 분리. 상한이 너무 낮으면 probe 개선이 먼저.
  리스크: EK100 probe는 ViT-G/384, Ego4D LTA probe는 ViT-L — 백본 교란. 1차는 ViT-L로 진행(교란 문서화),
  상한이 낮으면 ViT-G/384 probe를 Ego4D로 직접 학습하는 2차 검토.
- **Phase 1 — 파이프라인 이식**: 학습 코드(`train_grpo_action.py`)는 jsonl 핸드오프만 소비 → 무수정 재사용.
  ① `src/ego/datasets/ego4d.py` 어댑터(현재 TODO 스텁) ② EK100 하드코딩된 데이터 스크립트 7종
  파라미터화 — history cutoff는 처음부터 올바르게 ③ `configs/step2/f0_ego4d_wm_only.yaml` + 스크립트.
- **Phase 2 — 학습·검증**: Qwen2.5 + 동결 레시피 (EK100 기준 2×H200 ≈ 90분/500step).
  동일 지표 세트 + judge 곡선(gemini-2.5-pro). **belief 위축이 Ego4D에서 재현되면
  B0 필요성의 데이터셋-일반성 증거.**

### 10.4 역할 분담과 B0 핸드오프 물품

- **B0 (상대 담당자)**: hindsight projection DPO 구현·검증. 전달물 —
  ① F0 체크포인트 ② `<reasoning>/<task_belief>/<action>` 3구성 트레이스 포맷
  ③ `completion_log_every` 롤아웃 로그 (DPO 쌍 원료) ④ 수정된 memory 생성 규칙
  ⑤ **배터리 ③④ belief 인과성 baseline 수치**.
- **F0 Ego4D (우리)**: Phase 0→2.
- 방법론 논리 권고: F0의 선택적 위축(§6) ↔ B0의 belief DPO는 정확히 대응 —
  "F0가 GT 없이 WM 상한(0.374)까지 올리고, 상한을 넘는 능력(belief·history)은 B0가 만든다"로 서술.
  F-1 제외도 실측이 지지 (G1 자체가 작아 추가 RL 단계의 기대 수익 낮음).

## 11. 알려진 한계 · 열린 결정

1. memory가 GT 내레이션 유래 (리워드는 GT-free지만 입력은 아님). self-memory 실험 미실시.
2. held-out은 video-disjoint지만 참가자 동일(P01–P06) — 크로스도메인 아님.
3. EK100만 검증. Ego4D 이식 시 백본 교란(ViT-G vs ViT-L) 유의.
4. judge 곡선 표본 작음 (20지점×3생성×1종) — §6.1 Gemini 재채점으로 보강.
5. 학습·평가의 `--reward_mode` 불일치 금지 (프롬프트 분포가 어긋나면 숫자 무의미).
6. `max_steps`는 `save_steps` 배수여야 최종 체크포인트 생성.
7. **열린 결정 — 단계 명칭**: 논문용으로 F0/B0 → **Foresight / Hindsight**
   (Foresight Alignment / Hindsight Refinement) 제안 검토 중. B0의 핵심 기법명(hindsight projection)과
   정합적이고 HER 계보의 독법과도 일치. 확정 시 문서·코드 주석 일괄 반영.
